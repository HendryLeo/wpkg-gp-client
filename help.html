Einleitung
WPKG-GP Client ist ein grafisches Benutzerinterface was es normalen System Benutzer (Ohne Administatoren Rechte) ermöglich die Software auf ihrem System manuell zu aktualisieren. Der Prozess verläuft nach dem Start vollautomatisch im Hintergrund und der Benutzer wird über die Fortschritt der Aktualisierung informiert.
Geschrieben ist das Programm in Python/wxPython und nutzt für die eigentlich Software Aktualisierung WPKG-GP .

Update Benachrichtigungen
Das Programm überprüft in einem vom System Administrator festgelegten Interval ob Aktualisierungen für das System vorliegen und benachrichtigt den angemeldeten Benutzer.
Dabei wird jedoch nur nach neueren Versionen für bereits installierte Software-Pakete gesucht, Benachrichtigungen für anstehende Neuinstallationen gibt es (noch) nicht.
Da die Updates-Datenbank unabhängig von der realen Paket Datenbank auf dem Update-Server ist kann es unter Umständen dazu kommen, dass es für bereits anstehende Updates noch keine Benachrichtigungen gibt.
Über das Kontext-Menü in der Taskleiste kann über die Option Check for updates eine manuelle Suche gestartet werden.

System Aktualiserung
Per Doppelklick auf das Taskbar-Icon oder die Option Update System im Kotext-Menü wird das Aktualisierungs Fenster geöffnet. Von hier aus können Sie eine manuelle System Aktualisierung starten.

Achtung:
Schließen Sie Sicherheitshalber alle aktiven Anwendungen und speichern Sie ihre Dokumente da Programme ohne Vorwarnung während einer Aktualisierung geschlossen werden können. In Ausnahmefällen kann es sogar vorkommen, dass das komplette System ohne Benachrichtigung neustartet.

Der Aktuelle Fortschritt der Aktualisierung wird über einen Fortschrittsbalken und einem Textfeld (Aktueller Fortschritt)dargestellt, dieses stellt aber nur den Fortschritt des kompletten Aktualisierungs Vorgangs da und NICHT den Fortschritt der Installation der individuellen Software.
Der Prozess ist abgeschlossen wenn der Fortschrittsbalken komplett ist und unter Aktueller Fortschritt "WPKG Process Finished" erscheint. Über die Option "View Log" können Sie sich die durchgeführte Arbeiten im Detail aufführen lassen.

Fehlerbenachrichtigungen
Falls es bei der Aktualisierung des System Fehler bei der Installation einzelner Pakete gegeben hat wird nach dem Abschluss der Aktualisierung automatisch eine Fehlerprotokoll aufgerufen.

Nutzung im Universitäts Wlan
Um die Aktualisierung über das WLAN der Universät Hamburg durchzuführen wird eine VPN Verbindung per Cisco AnyConnect benötigt.
Die Software erkennt automatisch wenn eine VPN Verbindung besteht und blockiert Aktualisierungen für den AnyConnect Client, diese können nur manuell oder über eine Kabel Verbindung installiert werden.

Nutzung von Zuhause
Durch die Verwendung einer VPN Verbindung (Cisco AnyConnect) ist es theoretisch auch möglich das System außerhalb der Universität zu aktualisieren.
Davon wird jedoch AUSDRÜCKLICH abgeraten da größere Software Updates über eine Haushaltsübliche Internet Leitung sehr langsam sind und es im schlimmsten fall auch zu Verbindungsabbrüchen kommen kann was während einer Aktualisierung durchaus zu Systemfehlern führen könnte.
